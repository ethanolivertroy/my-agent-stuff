# Research: Official Linear MCP server for Pi extension parity (May 2026)

## Summary
Linear’s official MCP offering is a hosted remote MCP server, documented by Linear, intended to let MCP-capable clients act on a user’s Linear workspace through OAuth rather than a local STDIO package. For feature parity, implement against Linear’s GraphQL API/`@linear/sdk`, but treat exact MCP tool names/schemas as a live-contract item: Linear’s public docs appear to document connection/auth and capabilities, not a complete authoritative `tools/list` schema.

## Findings
1. **Authoritative entry point** — Use Linear’s MCP docs as the source of truth for the official server and client setup, and Linear’s developer docs for API behavior. Primary URLs: [Linear MCP docs](https://linear.app/docs/mcp), [Linear Developers](https://linear.app/developers), [GraphQL API](https://linear.app/developers/graphql), [TypeScript SDK](https://linear.app/developers/sdk), [OAuth 2.0 authentication](https://linear.app/developers/oauth-2-0-authentication).
2. **Transport** — The official server is remote/hosted at Linear rather than a package you run locally. Public examples/docs should be checked for the current endpoint, but the relevant hosted endpoints are expected to be the modern Streamable HTTP endpoint `https://mcp.linear.app/mcp` and/or legacy SSE endpoint `https://mcp.linear.app/sse`. MCP transport semantics come from the MCP spec; prefer Streamable HTTP for new Pi work and keep SSE only if Pi’s MCP bridge requires it. [Linear MCP](https://linear.app/docs/mcp), [MCP transports spec](https://modelcontextprotocol.io/specification)
3. **Auth model** — For the official hosted MCP server, plan for OAuth sign-in to Linear from the MCP client. For a Pi extension that calls Linear directly, use Linear OAuth for multi-user installs; personal API keys are suitable only for local/dev or single-user setups. Linear’s GraphQL endpoint is `https://api.linear.app/graphql`; API keys are sent in the `Authorization` header, while OAuth access tokens use bearer-token authorization. [OAuth docs](https://linear.app/developers/oauth-2-0-authentication), [GraphQL API](https://linear.app/developers/graphql)
4. **Exact MCP tool names** — I could not confirm an authoritative public list of the current official Linear MCP `tools/list` output. Do not assume client-displayed names such as `mcp__linear__...` are server tool names; clients often add those prefixes. Before claiming parity, run an authenticated MCP `initialize` + `tools/list` against Linear’s server and snapshot tool names, descriptions, and input schemas.
5. **Likely capability surface to mirror** — Public Linear MCP positioning and Linear API coverage imply parity should cover: read/list/search issues; get/create/update issues; add/list comments; list/get teams, users, workflow states, labels; list/get/create/update projects; list/get cycles; and read/create/update documents if exposed by the current API/server. These map naturally to Linear GraphQL queries/mutations and the generated TypeScript SDK. [GraphQL API](https://linear.app/developers/graphql), [SDK](https://linear.app/developers/sdk)
6. **API/SDK mappings for implementation** — Issue tools: GraphQL `issue`, `issues`, likely `issueSearch`, `issueCreate`, `issueUpdate`; SDK equivalents such as `LinearClient.issue()`, `LinearClient.issues()`, `createIssue()`, `updateIssue()`/model `.update()`. Comment tools: issue `comments`, `commentCreate`, `commentUpdate`. Project/cycle tools: `projects`, `project`, `projectCreate`, `projectUpdate`, `cycles`, `cycle`. Metadata: `teams`, `team`, `users`, `user`, `workflowStates`, `issueLabels`. Documents: verify current schema names, then use document queries/mutations exposed by the generated SDK. [GraphQL API](https://linear.app/developers/graphql), [`@linear/sdk` package](https://www.npmjs.com/package/@linear/sdk)
7. **Implementation details that affect parity** — Linear uses GraphQL connection pagination (`first`/`after`) and rich filters, so Pi tools should expose cursor pagination and filters instead of hard-coded limits. Mutating tools should preserve Linear semantics for team IDs, state IDs, assignee IDs, project IDs, labels, rich-text/Markdown descriptions, and issue identifiers like `ENG-123` versus UUIDs. [GraphQL API](https://linear.app/developers/graphql)

## Gaps
- Exact official tool names, descriptions, and JSON Schemas were not confidently discoverable from public docs alone. Next step: connect an MCP Inspector or Pi-compatible MCP client to `https://mcp.linear.app/mcp` with OAuth and capture `tools/list`.
- Endpoint details should be re-verified against Linear’s MCP page at implementation time, because remote MCP has been transitioning from SSE to Streamable HTTP.
- Document APIs and any newer Linear objects/features should be verified against the current GraphQL schema/SDK generation before building parity claims.
